Workflow Summary
PRE (1-2 days) → PHASE 1: Interview (15 min) → PHASE 2: Self-Study (2-3 days) → PHASE 3: Live Practice (3-4 hrs) → PHASE 4: Final Exam → PHASE 5: Probation (14 days)

Philosophy: The Chatting School teaches ALL theory (self-paced, 24/7). Live sessions are ONLY for practice, Q&A, and assessment. The Admin Dashboard tracks progress automatically.
Global Rules (Read First)
1. School = Primary Teacher — Recruits learn everything from the Chatting School. You do NOT re-teach theory in live sessions.
2. Fast Expulsion — Any recruit who fails to apply instructions, doesn’t improve with coaching, or breaks standards → removed immediately.
3. Voice Rule — During live sessions, microphones MUST remain open. Not interacting verbally = negative evaluation.
4. One-by-One Practice — During live practice, one recruit chats at a time. Others MUST watch closely and interact with HM when asked.
5. Admin Dashboard — Use it to monitor every recruit’s progress. Filter by group to see your current batch. Check quiz scores, completed modules, and section progress.
6. Invite Codes with Groups — Every batch gets its own invite code with a group name (e.g., “Batch-Mar-2026”). This lets you filter and track each batch separately.
7. Handovers — When ownership changes (HM → TL), communicate clearly in Telegram with full context.
8. Model Prep — Before each training group, verify the training model has ALL current scripts loaded from the Script Dashboard. If scripts are missing, coordinate with Script Manager before starting.
PRE

Objective

Filter applicants, give access to the school, and let them prove they can study independently. By the end of this phase, only candidates who passed Foundations + Tools should remain.

Mileh’s Actions

  • Filter applicants in Airtable → mark qualified as Interview
  • Send Discord invite → assign role: Applicant
  • Create thread for this group in #recruits
  • Go to Admin Panel → Invites: generate an invite code with group name (e.g., “Batch-Mar-2026”) and max uses (e.g., 10)
  • Post Welcome Message (below) with the invite code and 48h deadline
  • After 48h: check Admin Dashboard (filter by group) → who passed the Foundations Review Quiz (t-1)?
  • Passed t-1 → send Congrats + Interview scheduling link
  • Did not complete in time → remove from Discord
Admin Dashboard Tip: Go to Admin Panel → Dashboard tab → select your batch group in the dropdown. The funnel shows how many recruits completed each section. Hover over the bars to see names. If someone hasn’t started after 24h, send them a reminder.

PHASE 1

Interview (15 min)

Objective

Identify candidates with high proactivity, clear goals, A-player potential, and adequate English level. Verify they actually studied the School content (including new modules).

Pre-Interview: Check Admin Dashboard

Before each interview, open the Admin Panel and check the recruit’s profile:

  • Did they pass the Foundations Review Quiz (t-1)? What score?
  • How many modules in Tools did they complete?
  • Did they pass any other quizzes beyond the minimum? (shows initiative)
  • When did they register? How fast did they complete? (shows motivation)

Interview Structure (15 min)

  • “Tell me about yourself” (3 min): Background, experience, personal goals. This is the most revealing question — let them talk.
  • Motivation Check (3 min): Why do you want this job? What are your goals? What do you know about OnlyFans chatting?
  • Knowledge Check (7 min): Questions from Foundations + Tools:
    • What is OnlyFans? What does a chatter do?
    • Name 3 banned topics.
    • What is a PPV? What is a script?
    • What’s the difference between a new sub and a time waster?
    • What is the 48-hour rule?
    • What are the Vault Pro folder colors? What does red mean? Green?
    • What are Personal Info scripts? Why are they important?
    • What is a mass message? Name 3 tags you always exclude.
    • What is the purpose of a Positive Spin script?
  • Quick Writing Test (2 min): “Write me a short flirty message as if you were a model responding to a fan who said ‘Hey beautiful’.” — Evaluates English level AND personality.

Pass/Fail Criteria

  • ✔ Pass: Motivated, background shows initiative, answered knowledge questions correctly (at least 6/9), writing test shows B1+ English and personality
  • ❌ Fail: Can’t answer basics (didn’t study), writing test shows poor English or zero personality, unclear motivation, red flags in background

PHASE 2

Self-Study: Journey + Advanced (2-3 days)

Objective

The recruit completes all remaining school content independently. You monitor their progress and intervene only if they’re stuck or falling behind.

What the Recruit Studies

  • The Journey (7 modules): Tools Review Quiz, Rapport & Teasing Bridge, Sexting & PPV Drops, Aftercare & Re-engagement, Branch Rules & NR Waves, Fan Assessment & Prioritization, Annotated Chat Examples
  • Advanced (8 modules): Journey Review Quiz, Top 10 Objection Scripts, Custom Content Sales (TPDs + Fake Discount), Game Sequences, Multitasking, Shift Routine, Common Mistakes, Full Conversation Simulation
  • Go Live gate quizzes: Advanced Review Quiz (g-1)

Mileh’s Actions During This Phase

  • Daily: Check Admin Dashboard → filter by batch group → see who’s progressing
  • If stuck: If a recruit hasn’t progressed in 24h, send a reminder: “Hey, I see you haven’t completed any new modules today. Everything okay? Let me know if you’re stuck on something.”
  • If a recruit fails a quiz: They can retry (button says “Try Again”). If they fail 3+ times on the same quiz, they likely didn’t study — consider removing them.
  • When recruit passes g-1 (Advanced Review Quiz): Schedule the live practice session for the next business day.
  • Create the recruit’s Infloww account (if not done yet)
Admin Dashboard Tip: In the Students tab, click on a recruit’s name to see their full profile: which sections they’ve unlocked, every quiz score, and completion timestamps. Use this to assess effort and speed.

PHASE 3

Live Practice Session (3-4 hours)

Objective

Verify recruits can APPLY what they learned. This is practice and assessment only — not teaching. If a recruit can’t demonstrate basic knowledge, they did not study and should be removed.

Session Agenda

  • Verbal Knowledge Check (10 min): Fire rapid questions from all sections. Use the question bank — pick questions randomly. Anyone who can’t answer → warning.
  • Infloww Live Walkthrough (20 min): Screen share. Recruits follow along in their own Infloww accounts. Navigate: inbox, fan insights, vault, PPV history, scripts. Each recruit finds a fan profile and classifies as New Sub or Time Waster. Verify they check content colors (green/grey/red).
  • Script Dashboard Practice (10 min): Each recruit opens the Script Dashboard, finds a model’s guide, copies a Personal Info script, finds an objection script variant, uses the search bar to find a specific message ID.
  • Rapport + Teasing Practice (20 min): One-by-one: recruit writes Rapport and Teasing messages in-character. HM gives feedback on tone, length, personality match.
  • Top 10 Objections Drill (20 min): HM says the fan’s objection → recruit must respond with the correct script in-character. Fast-paced, no hesitation. Cover: /price, /discount, /free, /nomoney, /nosex, /real.
  • Custom Sales TPDs Practice (15 min): HM plays a fan asking for a custom → recruit executes full TPDs (Teasing → Price → Distraction Shift). Practice with 2-3 scenarios. Include Fake Discount technique.
  • Multitasking Scenario (15 min): Present 4 fans messaging at once. Recruit must: rank priority, write first response to each, explain reasoning.
  • Full Simulation (45 min, one-by-one): Each recruit does a complete conversation: New sub → Rapport → Teasing → Sexting → PPV → handle objection → NR wave → Aftercare → Custom pitch. HM plays the fan. Detailed feedback after each.
  • Shift Routine Review (10 min): Walk through start checklist + handoff checklist. Emphasize: “the next chatter should pick up without asking questions.”
HM Assessment Checklist — Live Practice
Recruit can navigate Infloww independently (inbox, fan insights, vault, PPV history)
Recruit can use the Script Dashboard (find scripts, copy, search)
Recruit identifies content colors correctly (green = paid, grey = seen, red = unpaid PPV)
Recruit classifies fans as New Sub or Time Waster using Fan Insights
Recruit handles objections using correct scripts without hesitation
Recruit executes TPDs correctly for custom sales (including Fake Discount)
Recruit demonstrates proper fan prioritization under pressure
Recruit completes full simulation with acceptable quality
Recruit stays in character throughout all exercises
Recruit participates actively (mic on, asks questions, engages)

PHASE 4

Final Exam

Objective

Validate comprehensive knowledge and ability to execute independently. Pass/fail gate before the supervised shift and probation.

Part 1: Written Quiz (15 min)

  • The recruit takes the Final Quiz (g-2) on the Chatting School website
  • 20 questions covering all sections: Foundations, Tools, Journey, Advanced
  • Topics: banned topics, journey phases, script selection, branch rules, fan prioritization, TPDs, pricing, shift routine, mass messages, content colors, personal scripts, Vault Pro
  • Pass: 80% (16/20 correct)
  • If fail: can retake ONCE (button says “Try Again”). Second fail = removed.
Admin Dashboard Tip: You can see the recruit’s quiz score in real time. Go to Students → click their name → see quiz results. The Final Quiz result will show immediately after they submit.

Part 2: Live Chatting Exam (1-2 hours)

  • Recruit chats in Infloww unassisted — handle real fans independently
  • HM observes but provides ZERO input, coaching, or intervention
  • Scoring rubric (each 1-5):
  • — Script adherence: Did they use scripts correctly?
  • — Character consistency: Did they stay in-model?
  • — Fan assessment: Did they prioritize correctly?
  • — Response speed: Under 2 minutes?
  • — Problem solving: How did they handle unexpected situations?
  • Pass: Average score 3.5/5 or higher, no category below 2

Mileh’s Post-Exam Actions

  • Passed: Update Airtable → Probation. Assign to a TL. Send Congrats message. Coordinate handover with Rycel via Telegram.
  • Failed: Explain specific failure points. Remove from Discord.

14 DAYS

Probation Period

Ownership Transfer: Mileh → Rycel + TL

After the exam, Mileh hands off to Rycel (Chatter Manager) who assigns the recruit to a Team Leader. From this point, the TL owns the recruit’s development.

  • Mileh sends handover to Rycel via Telegram (format above)
  • Rycel assigns to TL based on shift availability and team balance
  • TL takes over: coaching, monitoring, and pass/fail decision

Week 1: Close Monitoring

  • TL assigns 1-2 models (reduced workload)
  • TL reviews chats daily in the first 3 days
  • Coaching sessions: Day 2, Day 4, Day 6 of probation
  • Focus: script adherence, character consistency, reply speed
  • Recruit studies Ongoing Development modules in the school (Week 2: Advanced Objections, Week 3: Customs & Upselling)
  • Red flag triggers: consistent reply time >3 min, breaking character, ignoring scripts, banned topic incident

Week 2: Building Independence

  • TL increases workload to normal (full model assignments)
  • Coaching sessions: Day 8, Day 10, Day 12 of probation
  • KPI tracking begins: Sales/hr, Messages/hr, Reply Time, CVR
  • Focus: consistency, sales improvement, handling difficult situations
  • Recruit continues Ongoing Development modules (Week 4: Psychology & Mirroring, Week 5: KPI Mastery)

Day 14: Pass/Fail Decision

  • TL submits probation report to Rycel with: KPI averages, quality assessment, improvement trajectory, recommendation
  • Pass criteria:
  • — Sales/hr ≥ $20 (or clear upward trend)
  • — Reply time consistently < 2 min
  • — No banned topic violations
  • — Script adherence ≥ 70%
  • — Positive improvement trajectory
  • Fail criteria:
  • — Sales/hr < $15 with no improvement
  • — Repeated character breaks
  • — Ignoring TL coaching
  • — Banned topic violations
  • Rycel makes final call. If edge case → extend probation 7 days.
